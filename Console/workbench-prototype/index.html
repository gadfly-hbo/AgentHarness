pi-xanthil 资产迁移边界 (Migration Boundary)

Prototype planning view only; not a real migration scan.

Design Expression
Migrate to AgentHarness 后台能力开发 / 治理 / 实验 / 注册 / 分发
  • harness-packs/ → AgentHarness Packages / Capability Packs
  • server/tools/registry.ts 与 tools → Console Tool Registry / Tool Lifecycle
  • tool / skill 实验对比、治理、审计、策略 → Console 后台能力治理
  • onto-prompts, onto-validator, onto-export, onto-extract → OntoBase
  • knowledge-retrieval.ts 等知识检索能力 → KnowledgeBase
  • memory 维护、老化、去重、沉淀逻辑 → MemoryBase
Remain in pi-xanthil 上层产品应用 / 安装 / 使用 / 用户反馈
  • 面向最终用户的产品 UI 与业务任务入口
  • workspace、session、业务流程、安装和使用入口
  • skill 生成、tool 使用和前端反馈闭环
  • 与 pi-xanthil 产品体验强绑定的交互、配置和运行时编排
Contract / adapter boundary 权威 contract 与产品 adapter 分离

建议由 AgentHarness 逐步权威维护 agentharness-port.ts 的 canonical contract；pi-xanthil 保留 adapter/client，并通过显式消费契约接入能力。

Console 是 AgentHarness 的控制平面，不是 pi-xanthil 内部页面；它负责能力后台治理，pi-xanthil 负责产品侧使用体验。
